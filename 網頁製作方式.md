Roadmap — 確定「方向與優先順序」
 高層次的藍圖：目標、願景、要做哪些功能、先後順序與時程。回答「我們往哪走」。
PRD — 確定「具體要做什麼」
 把 Roadmap 的方向細化成明確的需求規格：每個功能要解決什麼問題、範圍邊界、驗收標準。回答「這個功能長什麼樣、做到什麼程度」。
Wireframe spec（線框圖） — 把需求「視覺化」
 寫程式前最關鍵的轉換點：將文字需求畫成畫面結構與排版，確認資訊架構與流程動線。屬於視覺化文件。
Claude Design — 寫程式碼前「最具體的成品」
 在動手寫 code 之前，先把畫面長相確定到最具體的程度，作為實作的依據。
Claude Code — 開始實際寫程式
 有了前四步的明確依據，才進入工程實作階段。
